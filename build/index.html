OR-100 Create order

  • Actors: Sales
  • Level: User goal
  • Mockup: Create order

Main success scenario:

  1. User enters order data.
  2. User enters order item data.
  3. System calculates the Item total for each item and Invoice total for the whole order.
  4. User saves the order.
  5. System validates the data.
  6. System creates new Order.

Extensions:

  • 2a. User wants to order more than 1 item:
    • 2a1. User adds more item rows and fills them in.
  • 2b. User wants to remove an item from the order:
    • 2b1. User deletes the item row.
  • 5a. A required field is empty:
    • 5a1. System displays error: Required.
  • 5b. Due date is in past:
    • 5b1. System displays error: Must be in future.
  • 5c. Item unit price or quantity is <= 0:
    • 5c1. System displays error: Must be > 0.
Refers to:  Sales,  Create order,  Order,  Order.dueDate,  OrderItem.unitPrice,  OrderItem.quantity
Referred from:  Sales,  Create order

OR-190 Delete order

  • Actors: Sales
  • Level: User goal
  • Mockup: Order detail

Main success scenario:

  1. User finds order in Order.status created.
  2. User deletes the order.
  3. System validates the data.
  4. System deletes all OrderItems of the order and the Order itself.

Extensions:

  • 3a. Order.status is not created:
    • 3a1. System displays error: Invalid status.
Refers to:  Sales,  Order detail,  Order.status,  Order item,  Order
Referred from:  Sales,  Order detail
